## 3. Page configuration model (within your framework)

Define the **EPK category** in your template config:

- `category: "EPK"`
- Default blocks list (with schema references):

1. `HeroBlock`
2. `MusicBlock`
3. `VideoBlock`
4. `BioBlock`
5. `StatsSocialProofBlock`
6. `TourDatesBlock`
7. `PressPhotosBlock`
8. `RiderBlock`
9. `ContactLinksBlock`

For each block, define a schema; example:

- `MusicBlock`:
    - `tracks: Track[]` where Track = { title, role (single / album / live), primary (bool), platformLinks: { spotifyUrl?, appleUrl?, soundcloudUrl?, bandcampUrl? } }.
- `PressPhotosBlock`:
    - `photos: Photo[]` where Photo = { title, thumbnailUrl, fullResUrl, orientation, credit }.

This keeps the EPK **config‚Äëdriven and extensible** while staying consistent with other page types.
